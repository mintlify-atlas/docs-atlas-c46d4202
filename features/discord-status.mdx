The `DiscordStatus` component (`components/discord-status.tsx`) fetches Gökhan's live Discord presence on page load and renders a small indicator in the header. It is used on both the homepage and the blog layout.

## How it works

1. On mount, the component calls `https://lantern.rest/api/v1/users/:userId` with the hard-coded user ID `957840712404193290`.
2. If the response status is `"offline"`, it stores `last_seen_at` and shows a "Last seen X ago" label using the `getRelativeTime` utility.
3. If the response status is anything other than `"offline"`, the component checks `active_platforms.spotify`. When a Spotify session is active, it renders a spinning album art thumbnail and the current track name and artist.
4. A pulsing green dot indicates online; a static gray dot indicates offline, loading, or error.
5. The entire widget links to `https://lantern.rest`.

## Component props

| Prop | Type | Default | Description |
|---|---|---|---|
| `userId` | `string` | — | Discord user ID to fetch presence for. |
| `hide` | `boolean` | `false` | Hides the component while still mounting it. Used in the blog layout to keep the Zustand store populated without showing the widget. |

## Zustand store

After a successful fetch, the raw `LanternUserResponse` object is written into the `useDiscordUserStore` Zustand store via `setData`. This lets other components on the page read the full presence data without making a second API call.

```typescript
import { useDiscordUserStore } from '@/stores/discord-user';

// Read the stored Discord user data from anywhere on the page
const data = useDiscordUserStore(state => state.data);
```

## TypeScript types

The response from `lantern.rest` is typed using the following interfaces from `types/index.d.ts`.

```typescript
export type LanternSpotify = {
  track_id: string;
  song: string;
  artist: string | string[];
  album: string;
  album_cover: string;
  start_time: {
    unix: number;
    raw: string;
  };
  end_time: {
    unix: number;
    raw: string;
  };
  time: {
    current_human_readable: string;
    total_human_readable: string;
  };
};

export type BaseUserType = {
  metadata: {
    id: string;
    username: string;
    discriminator: string;
    global_name: string | null;
    avatar: string | null;
    avatar_url: string | null;
    display_avatar_url: string;
    bot: boolean;
    flags: {
      human_readable: string[];
      bitfield: number | null | undefined;
    };
    monitoring_since: {
      unix: number | null;
      raw: Date | null;
    };
  };
  active_platforms: ClientPresenceStatusData;
  activities: (CustomStatusActivity | OtherActivity)[];
  storage: Map<string, string> | {};
};

export type LanternUserResponse =
  | (BaseUserType & { status: 'offline'; last_seen_at: { unix: number; raw: Date } })
  | (BaseUserType & { status: Exclude<string, 'offline'>; last_seen_at: { unix: number; raw: Date } });
```

## Analytics

When the widget is clicked, a [Rybbit](https://rybbit.io) custom event is fired.

```typescript
window.rybbit.event('discord_status_click');
```

<Note>
  The event fires only if the `rybbit` object is present on `window`. The link to `lantern.rest` always works regardless of whether analytics are enabled.
</Note>
